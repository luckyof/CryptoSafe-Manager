# CryptoSafe Manager

Безопасный менеджер паролей с открытым исходным кодом, построенный на принципах модульности и криптографической защиты. Проект развивается итерационно в рамках 8 спринтов, каждый из которых добавляет новый уровень функциональности и безопасности.

## Видение проекта

CryptoSafe Manager призван предоставить пользователям надёжное и удобное средство для хранения конфиденциальной информации (пароли, заметки, данные кредитных карт) с акцентом на безопасность, расширяемость и современные стандарты шифрования. Архитектура проекта спроектирована таким образом, чтобы каждый следующий спринт мог быть реализован без кардинальной переработки уже написанного кода.

## Дорожная карта спринтов

| Спринт | Название                      | Ключевые задачи                                                                 |
|--------|-------------------------------|---------------------------------------------------------------------------------|
| 1      | Фундамент                     | Модульная структура, база данных, заглушки шифрования, база GUI, события.      |
| 2      | Управление ключами            | Интеграция KeyManager, мастер-пароль, хранение соли и хеша в key_store.         |
| 3      | Реальное шифрование           | Замена XOR на AES-256-GCM, добавление аутентифицированного шифрования.          |
| 4      | Буфер обмена и таймеры        | Безопасная работа с буфером обмена, автоматическая очистка, таймеры неактивности. |
| 5      | Журнал аудита и целостность   | Полноценный аудит действий, цифровая подпись записей аудита.                    |
| 6      | Теги и поиск                  | Категоризация записей, полнотекстовый поиск, фильтры.                           |
| 7      | Многопользовательность и блокировка | Поддержка нескольких профилей, автоматическая блокировка при простое.          |
| 8      | Резервное копирование и экспорт| Создание резервных копий, экспорт/импорт данных, финальная сборка.              |

## Архитектура (MVC)

Проект следует паттерну Model-View-Controller (MVC), что обеспечивает чёткое разделение ответственности и упрощает тестирование и расширение.

```
┌─────────────────────────────────────────────────────────────┐
│                         Пользователь                         │
└───────────────────────────────┬─────────────────────────────┘
                                │
                                ▼
┌─────────────────────────────────────────────────────────────┐
│  GUI (View)                     ┌─────────────────────────┐ │
│  - main_window.py                │   Виджеты (widgets/)   │ │
│  - диалоги (settings_dialog,     │ - PasswordEntry        │ │
│    setup_wizard)                  │ - SecureTable          │ │
│                                 │ - AuditLogViewer        │ │
└─────────────────┬───────────────┴─────────────────────────┘ │
                  │              ▲                              │
                  │ взаимодействие│ (события)                    │
                  ▼              │                              │
┌─────────────────────────────────────────────────────────────┐ │
│  Controller / Core              ┌─────────────────────────┐ │ │
│  - events.py (EventBus)          │ - config.py            │ │ │
│  - state_manager.py               │ - audit.py             │ │ │
│  - key_manager.py                 │ - crypto/*             │ │ │
└─────────────────┬───────────────┴─────────────────────────┘ │ │
                  │              │                              │
                  │     запросы  │  уведомления                 │
                  ▼              ▼                              │
┌─────────────────────────────────────────────────────────────┐ │
│  Model (Database)                                            │ │
│  - db.py (DatabaseHelper)                                    │ │
│  - таблицы: vault_entries, audit_log, settings, key_store   │ │
└─────────────────────────────────────────────────────────────┘ │
                                                                │
└───────────────────────────────────────────────────────────────┘
```

**Основные компоненты:**

- **View (GUI)** — отвечает за отображение и взаимодействие с пользователем. Содержит главное окно, переиспользуемые виджеты и диалоги.
- **Controller (Core)** — содержит бизнес-логику: управление событиями, конфигурацией, состоянием приложения, криптографическими операциями.
- **Model (Database)** — обеспечивает работу с базой данных SQLite, создание таблиц, выполнение запросов.

## Установка и настройка

### Предварительные требования

- Python 3.9 или выше
- Git (для клонирования репозитория)

### Инструкция по установке

1. **Клонируйте репозиторий**

   ```bash
   git clone https://github.com/yourusername/cryptosafe-manager.git
   cd cryptosafe-manager
   ```

2. **Создайте виртуальное окружение**

   ```bash
   python -m venv venv
   ```

   Активируйте его:
   - **Windows:** `venv\Scripts\activate`
   - **Linux/macOS:** `source venv/bin/activate`

3. **Установите зависимости**

   На данный момент проект использует только стандартные библиотеки Python. Для запуска тестов требуется `pytest`. Установите его командой:

   ```bash
   pip install pytest
   ```

   *Примечание:* В дальнейшем список зависимостей будет расширяться и фиксироваться в `requirements.txt`.

4. **Запустите приложение**

   ```bash
   python -m src.main
   ```

   При первом запуске откроется мастер первоначальной настройки, где нужно будет указать путь к базе данных и задать мастер-пароль.

### Запуск тестов

Для проверки работоспособности основных модулей выполните:

```bash
pytest tests/
```

## Использование

После успешного запуска откроется главное окно программы. На данный момент реализованы:

- Меню «Файл», «Правка», «Вид», «Справка».
- Центральная таблица для отображения записей (заглушка).
- Строка состояния с индикатором статуса.
- Диалог настроек (заглушка).
- Мастер первоначальной настройки.
- Журнал аудита (просмотр тестовых записей).

Все операции с данными (добавление, редактирование, удаление) находятся в стадии заглушек и будут реализованы в следующих спринтах.
