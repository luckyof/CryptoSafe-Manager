# CryptoSafe Manager

Безопасный менеджер паролей с открытым исходным кодом, построенный на принципах модульности и криптографической защиты. Проект развивается итерационно в рамках 8 спринтов, каждый из которых добавляет новый уровень функциональности и безопасности.

## Видение проекта

CryptoSafe Manager призван предоставить пользователям надёжное и удобное средство для хранения конфиденциальной информации (пароли, заметки, данные кредитных карт) с акцентом на безопасность, расширяемость и современные стандарты шифрования. Архитектура проекта спроектирована таким образом, чтобы каждый следующий спринт мог быть реализован без кардинальной переработки уже написанного кода.

## Архитектура (MVC)

Проект следует паттерну Model-View-Controller (MVC), что обеспечивает чёткое разделение ответственности и упрощает тестирование и расширение.

**Основные компоненты:**

- **View (GUI)** — отвечает за отображение и взаимодействие с пользователем. Содержит главное окно, переиспользуемые виджеты и диалоги.
- **Controller (Core)** — содержит бизнес-логику: управление событиями, конфигурацией, состоянием приложения, криптографическими операциями.
- **Model (Database)** — обеспечивает работу с базой данных SQLite, создание таблиц, выполнение запросов.

## Установка и настройка

### Предварительные требования

- Python 3.9 или выше
- Git (для клонирования репозитория)

### Инструкция по установке

1. **Клонируйте репозиторий**

   ```bash
   git clone https://github.com/yourusername/cryptosafe-manager.git
   cd cryptosafe-manager
   ```

2. **Создайте виртуальное окружение**

   ```bash
   python -m venv venv
   ```

   Активируйте его:
   - **Windows:** `venv\Scripts\activate`
   - **Linux/macOS:** `source venv/bin/activate`

3. **Установите зависимости**

   Проект использует внешние зависимости для криптографии и проверки мастер-пароля. Установите их командой:

   ```bash
   pip install -r requirements.txt
   ```

   Основные зависимости:
   - `argon2-cffi`
   - `cryptography`
   - `pytest`
   - `pyperclip`
   - `pywin32` на Windows
   - `pyobjc-framework-Cocoa` на macOS

   Для Linux-буфера обмена может потребоваться один из системных backend-пакетов: `wl-clipboard`, `xclip` или `xsel`.

4. **Запустите приложение**

   ```bash
   python main.py
   ```

   При первом запуске откроется мастер первоначальной настройки, где нужно будет указать путь к базе данных и задать мастер-пароль.

### Запуск тестов

Для проверки работоспособности основных модулей выполните:

```bash
pytest tests/
```

Для запуска только тестов Sprint 3:

```bash
pytest tests/test_sprint3.py
```

Для запуска только тестов Sprint 4:

```bash
pytest tests/test_sprint4_clipboard.py
pytest tests/test_sprint4_config.py
pytest tests/test_sprint4_validation.py
pytest tests/test_sprint4_integration.py
pytest tests/test_sprint4_performance.py
pytest tests/test_sprint4_security.py
pytest tests/test_sprint4_resilience.py
```

## Использование (1 спринт)

После успешного запуска откроется главное окно программы. В Sprint 1 были реализованы базовые элементы интерфейса:

- Меню «Файл», «Правка», «Вид», «Справка».
- Центральная таблица для отображения записей.
- Строка состояния с индикатором статуса.
- Диалог настроек.
- Мастер первоначальной настройки.
- Журнал аудита.

## Использование (2 спринт)

Во втором спринте были добавлены:

- Работа с мастер-паролем.
  Мастер-пароль никогда не хранится в явном виде.
  Для проверки используется соль + хеш Argon2id.
  При вводе неверного пароля доступ к данным блокируется.

- Защищённая память.
  Ключи шифрования хранятся в заблокированной памяти (через `mlock` на Linux/macOS или `VirtualLock` на Windows).
  Это предотвращает выгрузку чувствительных данных в swap-файл.

- Атомарные транзакции.
  Операции смены пароля выполняются атомарно: либо полностью, либо с откатом.
  Это гарантирует целостность данных даже при сбое во время обновления.

## Использование (3 спринт)

В третьем спринте реализованы основные рабочие функции хранилища:

- Реальное шифрование записей через AES-256-GCM.
- CRUD-операции для записей хранилища.
- Мягкое удаление записей с переносом в `deleted_entries`.
- Генератор паролей с оценкой сложности.
- Поиск по полям `title`, `username`, `url`, `notes`, `category`.
- Field-specific фильтры вроде `category:"Work"`.
- Фильтрация по тегам.
- Фильтрация по диапазону дат (`created_at`, `updated_at`).
- Фильтрация по силе пароля.

Дополнительно:

- Ошибки доступа к записи не раскрывают существование конкретного `entry_id`.
- Реализован аудит ключевых операций с записями.
- Главное окно работает с реальными данными, а не с заглушками.

## Использование (4 спринт)

В четвёртом спринте добавлена защищённая работа с системным буфером обмена:

- новый модуль `src/core/clipboard/` с сервисом clipboard, платформенными адаптерами и монитором;
- кнопки и контекстное меню для копирования логина, пароля и данных записи;
- автоматическая очистка буфера обмена по таймеру;
- очистка при блокировке, выходе, закрытии приложения и замене содержимого;
- индикатор состояния буфера в строке статуса и подсветка записи, чьи данные находятся в буфере;
- masked preview содержимого с раскрытием через повторную аутентификацию;
- мониторинг внешнего изменения буфера и подозрительной активности;
- ускоренная очистка и возможность блокировки копирования после подозрительного события;
- настройки таймера, уведомлений, профилей безопасности и мониторинга;
- поддержка Windows/macOS/Linux backend-адаптеров и fallback через `pyperclip`;
- защита данных в памяти: XOR-маска, попытка `VirtualLock`/`mlock`, зануление после очистки;
- интеграция с `EntryManager`, audit log и future hooks для TOTP, secure sharing и panic mode.

Добавлены тесты Sprint 4 для clipboard-архитектуры, конфигурации, валидации, интеграции, производительности, security requirements и error handling.
